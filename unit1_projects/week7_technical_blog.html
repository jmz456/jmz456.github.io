SQL Injection

What is an Injection threat and how serious is it?

An SQL injection is a form of attack against websites. More specifically, “It’s a code injection technique used to attack data-driven applications in which malicious SQL statements are inserted into an entry field for execution.”( source: http://en.wikipedia.org/wiki/SQL_injection) In other words, an SQL injection can destroy your database.

So how does it work? The following is an example if you didn’t take the necessary precautions. Let’s say a user with malicious intentions for whatever reasons decides to make havoc of your database. This user may search for methods of data input like a username or userpass field. Maybe it’s a search for a customer number or a field for an employee id.

Now from your perspective, let’s say you have this input point on your website. You would take this user’s input and have it append to a customized SQL string, and execute that SQL. However, when the malicious user puts in a code that can interact with your SQL string, depending on how your string was written, the user might try to include a piece of code that could finish off your SQL string. Here’s an example of what that might look like:

Your website contains the following input point:

Customer ID : x’; DROP TABLE Orders; -- . Submit .

You have custom string that looks like the following:

sqlString = "SELECT * FROM orders WHERE CustomerID = '" +textbox.value + "'";

This is what gets put into your string and the following is executed.

sqlString = "SELECT * FROM Orders WHERE CustomerID = 'x'; DROP TABLE Orders; --'"

executSQL(sqlString);

Now, what has happened above is the attacker has attempted to end the first clause and create a new one separated by the semicolon. So instead of running one line of code you could potentially run 2 or more. The DROP TABLE statement is used to delete a table which would put you in a compromised position. Now you may not have a table called Orders, but it could likely be something easy to figure out after multiple attempts.

So what can you do to prevent such an attack. As you progress, learn to create input validation techniques and set rules for input parameters like the type of input or the type of syntax.

Limit the number of privileges that users have when accessing a database.

As you become more advanced with sql queries, you will eventually want to start saving your queries. Ideally, you’re going to want to create a stored procedure . It’s like a method in Ruby where you can write out the code, give it a name and store it within your database. Then you can call it and have it executed any number of times. By writing stored procedures as apposed to standard lines of queries you can increase protection from SQL injection.
