Functional Programming

I haven't paid attention to functional programming before, but I think I will now.

At the base of Functional programming is the mathematical notion of function composition. “Composing functions f and g means g(f(x)) – f’s output becomes g’s input.” (Erkki Lindpere) It’s a programming paradigm that models computations as the evaluation of expressions.

You could think of it as a mathematical approach to writing code which is the way it started and has done so for the last 40 some years.

Object-oriented programming (OOP) is a programming language model organized around objects rather than "actions" and data rather than logic. It’s intention is to make thinking about programming closer to thinking about the real world.

A great piece I found on code.google.com really helps to get a visual of the sense. Applying functional programming to Ruby. Just a few examples.

Don't update variables

Don't update them, just create new ones.

Don't append to arrays or strings

No:

indexes = [1, 2, 3]

indexes << 4

indexes # [1, 2, 3, 4]

Yes:

indexes = [1, 2, 3]

all_indexes = indexes + [4] # [1, 2, 3, 4]

Don't use bang methods which modify in-place

No:

string = "hello"

string.gsub!(/l/, 'z')

string # "hezzo"

Yes:

string = "hello"

new_string = string.gsub(/l/, 'z') # "hezzo"
